## YOUR OBJECTIVES

### 1. Product Clarity Audit
- What problem is this solving?
- Is the "Top of the Mountain" clear?
- Are user outcomes explicitly defined?
- Are there missing user journeys or edge cases?

Call out:
- Vagueness
- Missing actors
- Unclear success criteria

---

### 2. Conceptual Integrity Audit
- Does the spec define clear domain concepts?
- Are terms consistent and reusable?
- Are there implicit concepts that should be explicit?

Call out:
- Naming inconsistencies
- Missing domain model
- Leaky abstractions

---

### 3. Architecture & System Design Audit
- Is there a clear mental model of the system?
- Are responsibilities well separated?
- Are boundaries (contexts, agents, modules) defined?

Call out:
- Tight coupling
- Missing components
- Unclear data flow
- Violations of single responsibility

---

### 4. Agent Execution Readiness (CRITICAL)
This spec will be executed by AI agents.

Evaluate:
- Can an agent implement this WITHOUT guessing?
- Are steps deterministic and testable?
- Are inputs/outputs clearly defined?

Call out:
- Missing schemas
- Undefined interfaces
- Implicit logic
- Areas where agents will hallucinate or diverge

---

### 5. State & Workflow Integrity
- Are states clearly defined (e.g., Pending → Active → Done)?
- Are transitions explicit?
- Are failure modes handled?

Call out:
- Missing states
- Ambiguous transitions
- No rollback / recovery paths

---

### 6. Data & Storage Model
- What data exists?
- Where does it live?
- How is it retrieved and updated?

Call out:
- Missing persistence strategy
- Undefined schemas
- Lack of temporal handling (important for evolving systems)

---

### 7. Entropy & Maintenance Risk
- Where will this system decay over time?
- What will become inconsistent?
- What will generate dead code or duplicated logic?

Call out:
- Overly flexible areas
- Lack of constraints
- Missing cleanup or lifecycle rules

---

### 8. Security & Safety Considerations
(especially important if user data or external input is involved)

Call out:
- Prompt injection risks
- Data leakage risks
- Over-permissioned actions
- Unsafe automation

---

### 9. Missing Sections / Improvements

Explicitly list:
- Sections that should exist but do not
- Things that should be expanded
- Things that should be removed or simplified
