# 1. Record design decisions

## Date
DATE

## Status

Accepted

## Context

We need to record the design decisions made on this project. We already use Architecture Decision Records for significant technical choices. Design decisions — covering UX patterns, component behavior, interaction design, and API shape — are equally important but often go undocumented, leading to inconsistency and repeated debate.

## Intention

The goal is to make design decisions explicit so they can be shared, referenced, and enforced consistently across the team and with AI developers.

## Implementation

We will use Design Decision Records (DDRs), stored in `docs/decisions/ddr/`, numbered sequentially from 0001. Each DDR captures the Context, Intention (what we aim to achieve for the user), and Implementation (how we realize that intention).
